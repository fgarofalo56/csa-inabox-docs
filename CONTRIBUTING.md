# Contributing to CSA in-a-Box Documentation

Thank you for your interest in contributing to the Cloud Scale Analytics in-a-Box documentation!

Please refer to the comprehensive [Contributing Guide](./docs/guides/CONTRIBUTING_GUIDE.md) for detailed contribution instructions, including:

- How to set up your development environment
- Documentation standards and style guide
- Pull request process
- Code review guidelines

## Quick Start

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes following our [Markdown Style Guide](./docs/guides/MARKDOWN_STYLE_GUIDE.md)
4. Run linting: `markdownlint '**/*.md'`
5. Submit a pull request
