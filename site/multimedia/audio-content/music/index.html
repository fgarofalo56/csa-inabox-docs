    Usage Guidelines¶

    Volume Levels¶

    Narration:  -18 dB (primary)
Music:      -30 dB (background)
SFX:        -24 dB (effects)

    Mixing Tips¶

    1. Ducking: Lower music volume when narration occurs
    2. Fade In: 2-3 seconds at video start
    3. Fade Out: 3-5 seconds at video end
    4. Loop Points: Ensure seamless loops for extended use

    Audio Processing¶

    EQ Settings (Music):
- High-pass filter: 100 Hz (remove sub-bass)
- Reduce: 2-4 kHz (make room for voice)
- Gentle compression: 3:1 ratio

Normalize to: -3 dB peak

    File Specifications¶

    Format Requirements¶

    • Format: WAV (lossless) or MP3 320kbps
    • Sample Rate: 48 kHz
    • Bit Depth: 24-bit (WAV) or 320kbps (MP3)
    • Channels: Stereo

    File Naming Convention¶

    [Category]_[TrackName]_[Duration]_[Tempo].[ext]

Examples:
Corporate_AzureAmbient_3m00s_90bpm.wav
Upbeat_InnovationDrive_2m45s_130bpm.mp3
Ambient_FocusFlow_10m00s_80bpm.wav

    Music Selection Checklist¶

    Before using a track, verify:

    • [ ] License allows commercial use
    • [ ] Attribution requirements documented
    • [ ] Duration appropriate for content
    • [ ] Tempo matches video energy
    • [ ] No copyrighted samples
    • [ ] Quality meets technical specs
    • [ ] Loops seamlessly (if needed)
    • [ ] Ducking automation tested
    • [ ] Volume levels appropriate

    Audio Editing Tips¶

    Creating Loops¶

    1. Find natural loop point (usually end of phrase)
2. Add 1-2 second crossfade
3. Test for seamless transition
4. Export as separate loop file

    Shortening Tracks¶

    1. Identify chorus/main theme
2. Remove verses or repetitions
3. Ensure musical coherence
4. Fade in/out appropriately

    Creating Variations¶

    1. Duplicate original track
2. Adjust tempo (10% max)
3. Apply slight EQ changes
4. Create "lite" version (fewer instruments)

    Accessibility Considerations¶

    • Provide audio descriptions where needed
    • Ensure music doesn't interfere with narration
    • Support closed captions for all audio
    • Test with hearing-impaired users
    • Provide volume controls in interactive content
